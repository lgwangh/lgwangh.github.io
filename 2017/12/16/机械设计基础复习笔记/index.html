我们老师真是深知学生的心思，不考的东西几乎都没讲(●ˇ∀ˇ●)不过这也导致这份笔记内容不是很全面。

关于笔记的撰写

之前就提过，Markdown的结构太简单，导致方言泛滥，着实恼人。前几天读python的help文档，发现他们似乎使用了某种标记语法，查了查，发现这就是reStructedText。这种标记语法是python帮助文档的标准，而有一个库Sphinx可以将.rst格式的文件轻松转化成赏心悦目的文档。那还等什么，管他什么复习，先用起来再说。

然而，事情总是不会那么轻松，配置地狱永远张开大嘴欢迎我，但这么多内容放在这个“机械设计”的标题底下实在是跑题，我应该在另外一篇博文中详说一下。简而言之，就是使用.rst + Sphinx + git + github + readthedocs的组合操作。
