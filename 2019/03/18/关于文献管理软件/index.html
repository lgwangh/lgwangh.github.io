之前一直在用学校买的NoteExpress，还凑活事。然而自打把这破软件推荐给同学之后，它就出毛病了，全文下载也不能用，浏览器插件也挂了。

所以现在只能找个新的。理想的软件是这样的：

  • 不要钱
  • 对中文文献有良好支持（太菜了，还需要看中文文献）
  • 最多只需要用到浏览器一次，也就是说，要么浏览器保存题录→软件全文下载，要么浏览器下载PDF→软件从中识别题录，总之不能两者都需要下载
  • 能够内部打开PDF，并将阅读时做的笔记整理出来——或者能全文搜索也行，我不想一篇文献看好多遍才找得到要引用的内容
  • 能够导出.bib文件，因为已经决定了主要采用\(\rm{\LaTeX}\)撰写论文

于是测试了众多软件。

测试结果

软件 问题 结论
NoteExpress 不符合一次原则；不能与PDF笔记同步 ×
EndNote 没钱 ×
Qiqqa 笔记功能挺有意思的，然而内建阅读器不支持中文显示 ×
CNKI E-Study 对中文支持很好，然而不能导出.bib ×
Citavi 浏览器插件不能用，因而不能满足一次原则；其实PDF阅读、笔记功能很全，英文文献能全文下载，可惜了 ×
JabRef Java铸就，奇卡无比，险些死机，敬而远之 ×
Mendeley 对中文文献不符合一次原则 ？
Zotero 看似简陋，实际还行？居然能下载国外文献，插件居然在知网也能用，可惜不能与PDF笔记同步 ？
ReadCube 咋没有免费版了呢？$3/月倒也不算贵 ？

鉴于目前用到的中文文献较多的情况，可能就先使用Zotero了Orz。他年我若成大佬，报与Mendeley一处开……

Zotero配置与插件

基本设置

在编辑 -> 首选项中：

  • 常规：相比SumtraPDF，把默认PDF改成PDFXchange或Acrobat会比较方便注释。
  • 同步：注册个账号，其它详见下一节。
  • 引用：获取更多样式中可以找到GBT7714-2005，没有2015版，但是鉴于我使用\(\rm\LaTeX\)，此功能少用，忍了。并且可以自己用CSL语言写一个样式，这里是编辑器。
  • 高级 -> 文件和文件夹：鉴于打算使用文件本身作为附件（而不是链接），不填链接附件的根目录。数据存储位置设置成自定义的位置。

云端同步

Zotero自带的同步空间较小，这里提供了多种数据同步方案。其中WebDAV法和同步盘+ZotFile法各有优劣：

优势 劣势
WebDAV 配置容易，无需开启网盘客户端，删除题录即可删除附件 文件结构差，都是8位随机字符的文件夹
同步盘+ZotFile 文件结构清晰，可把所有附件放在同一文件夹下 需要常开网盘客户端，由于是文件链接，删除题录附件还在，占空间

最终选择了坚果云的WebDAV，配置方法见http://help.jianguoyun.com/?p=3168。使用这种方法无须启动坚果云，减少了内存的占用和开机时间。不过这种方法导致文件夹名称看不懂，很难直接从网盘中读取附件，所以安卓端需要额外的软件：\(\text{ZotEz}^2\)。启动软件时需要选择云端数据库，在这里不需要选WebDAV项，直接用Zotero账号登录即可。

知识结构

阳志平的网志中提供了一份较为全面的Zotero教程，他在其中提到：

作为对Tag系统的改良。在Zotero中，我们不推荐将Tag作为分类使用，而更多当做【印象】使用！请记住，Tag始终会随着时间而动荡甚至走向崩溃！所以，更重要的是，你当时当下对该文档的【印象】，而不是【分类】！

那么，如何更好分类？

  • 以基本属性作为分类，基本属性是你可以很快归类的，不太会随时间动荡的；
  • 再使用搜索代替随时间动荡的内容；
  • 不再将Tag作为【分类】属性；而是当下【印象】工具。

Zotero重知识管理的三重结构：

  • 分类：很简单，点击新建分类即可
  • 搜索：右键选择我的文库，然后新建搜索即可定义并保存搜索，相当于会自动更新的文件夹
  • 标签：默认在导入题录时会按照关键字自动加tag，不希望的话可以在编辑 -> 首选项 -> 常规 -> 杂项中改掉。可以给标签赋予颜色和快捷键（最多9个）：在标签云中右键选择标签，选择“指派颜色”即可。具有颜色的标签会表现为标题前的色块，十分醒目。我目前设置的标签有：未读、重要、需引用。

插件

需要先从网上下载到.xpi文件，然后在工具 -> 插件选择安装。

ZotFile

自动重命名附件，自动移动位置，还能提取PDF中的注释，很有用，下载地址。

第一项是选择附件源文件夹，也就是浏览器默认下载目录。之后在右键菜单选择Add Attachment from Folder时，会自动检测到该文件夹中最新更改的文件。另外根据介绍还有自动监视文件夹的功能，但是没有发现……第二项是附件保存位置，根据之前同步方式的分析，保持设置即可，这样附件会移动至之前首选项中选择的位置。

Markdown Here

Zotero的笔记编辑器不能用Markdown语法，不太方便，幸好浏览器插件Markdown Here是可以移植过来的，并且有Zotero论坛大佬做好了插件包。

安装后，在笔记编辑器可以用Ctrl + Alt + m切换plain/render模式。注意，由于并非完全适配，不要在render模式下编辑，否则再Ctrl + Alt + m一下之后新增内容无法保存。

其他

  • Better BibTex for Zotero：号称能改善\(\rm\LaTeX​\)输出，还没试过。
  • ZoteroQuickLook：为Zotero提供Mac的QuickLook般的PDF预览功能，在Windows上需要先安装QuickLook（不必开机启动，用到时会自动启动）。
  • Zotero Storage Scanner：号称能查找重复文献之类，还没试过。
  • Zutilo Utility for Zotero：为Zotero增加一些有用的右键菜单和快捷键。
